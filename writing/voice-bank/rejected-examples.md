# Rejected examples

## Rejected: tidy antithesis

Surface: x-research
Reason: juxtaposition pivot

Text:
```text
OpenClaw's real pitch isn't privacy. It's control. Running agents on hardware you own means the API costs stop when you stop. That's the thing cloud-hosted AI can't match.
```

Why it failed:
- Uses the banned "real pitch isn't X. It's Y." structure.
- Reads like a model announcing insight.
- The useful detail appears after the AI skeleton.

Better direction:
```text
Running agents on hardware you own has one very boring advantage: the API costs stop when you stop.
```

## Rejected: dashboard antithesis

Surface: x-research
Reason: tidy antithesis and abstract nouns

Text:
```text
The real operator tooling market isn't dashboards. It's the invisible stuff: trace visualization, latency attribution, and context injection when an agent veers off-script.
```

Why it failed:
- Uses the same "real X isn't A" skeleton.
- "Invisible stuff" sounds like a setup line.
- Better concrete detail exists later in the sentence.

Better direction:
```text
Trace visualization, latency attribution, and context injection are the boring parts that decide whether an agent survives contact with production.
```

## Rejected: tell-tale AI connective

Surface: x-research
Reason: Angel called this tell-tale AI voice

Text:
```text
That gets messy when data is fragmented across platforms.
```

Why it failed:
- Starts with a generic connective.
- Uses abstract nouns: data, platforms.
- Could fit hundreds of sources.

Better direction:
```text
Things like WhatsApp history, tweet archives, or bookmarks you might need years from now, that's where it gets messy.
```

## Rejected: polished antithesis closer

Surface: x-research
Reason: too tidy, too copywritten

Text:
```text
Solo builders should treat the prompt as a work order, not a motivational speech.
```

Why it failed:
- The antithesis is neat enough to smell generated.
- Angel prefers direct colon framing for this idea.

Better direction:
```text
Solo builders should treat the prompt as a work order: Name the files, name the constraints, name the stop condition.
```
